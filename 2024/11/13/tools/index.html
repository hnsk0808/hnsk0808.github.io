Github

Github提交PR

  1. fork目标库
  2. 在fork库中，新建branch
  3. 在新branch中，修改源代码 你的修改应该在一个fork库的新branch里
  4. 提交pr，把fork库的新branch提交到上游 源库的main或者dev分支
  5. 如果pr不够完善，你仍可修改你的branch pr里存了你的branch，所以pr会同步你的新commit
  6. reviewers可能会修改你发布的branch
  7. 管理员merge/拒绝你的pr
